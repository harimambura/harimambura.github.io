Форма

<form action="" method="POST" novalidate autocomplete="on" target="_top" enctype="application/x-www-form-urlencoded"> <input type="text" name="" id=""> <input type="button" value="Кнопка"> <br> <input type="submit"> </form>


Описание пареметров form: action - указывает, на какую страницу будут отправлены данные, target - куда будет выведен результат отправки данных (можно указать фрейм), novalidate - значит, что HTML5-проверка правильности введенных данных произведена не будет, autocomplete - значит, что при обновлении страницы input'ы будут заполнены данными, введенными ранее. Значения on или off.
enctype - указывает тип кодирования, значения: application/x-www-form-urlencoded - будет заменять русские буквы на хекс значения, а пробелы на +; multipart/form-data - для того, чтобы отправить файлы (не будет кодирования данных)

Текстовые input'ы

Текстовое поле: (<input type="text">)
Поле пароля: (<input type="password">)
Поле email: (<input type="email">)
Поле телефона: (<input type="tel">)
Поле поиска: (<input type="search">)
Каждое предназначено для сових нужд, в каждое вводится текст, поле пароля скрывает введенные символы. К каждому полю применим параметр maxlength, в котором указывается, сколько символов можно ввести. Каждому текстовому полю можно задать атрибут autocomplete по аналогии с формой. Атрибут placeholder задает текст по умолчанию в поле, при нажатии\вводе текста значение по умолчанию пропадает(placeholder="write here..."). Атрибут required указывает, что поле обязательно для заполнения, параметров не имеет. Если в форме есть novalidate, то это атрибут не работает. Атрибут readonly делает инпут только для чтения. size устанавливает размер. spellcheck - проверка орфографии введенного текста, принимает значения true/false/default. pattern - для установки шаблона вводимого текста, ?. Так же атрибуты id и name, по имени будут отправляться данные на сервер. autofocus указывает, что при загрузке страцицы фокус будет на данном инпуте, tabindex - порядок табуляции, значения - порядковые номера.

Атрибут name у инпутов обязателен, для отправки данных. Для того, чтобы сделать группу радио-кнопок, необходимо задать им одинаковый атрибут name (как в примере). Атрибут value должен содержать значение, которое будет отправлено на сервер. Атрибут name у инпутов обязателен, для отправки данных. Для того, чтобы сделать группу радио-кнопок, необходимо задать им одинаковый атрибут name (как в примере). Атрибут value должен содержать значение, которое будет отправлено на сервер.
Инпут диапазон: имеет атрибуты min и max, которые задают минимальное и максимальное значение, так же step - указывает шаг.
Инпут выбора файлов: имеет артибут accept, указывающий, какой тип файлов выбирать, значением является один или несколько MIME-типов, перечилсенных через запятую, либо значения audio/*, video/*, image/*. Атрибут multiple - указывает, что можно выбрать несколько файлов, значений не имеет.
Meter: при заданных артибутах low и high меняет цвет в пределах диапазона, заданного этими атрибутами, если ближе к low - зеленый, ближе к high - желтый, если больше - красный.

Инпуты кнопки

Просто кнопка: (<input type="button">)
Графическая кнока отправить: (<input type="image">)
Кнопка очистки формы: (<input type="reset">)
Кнопка оптравки формы: (<input type="submit">)
Кнопка button используется в JS, никаких действий по умолчанию не выполняет, для работы необходимо задать name , для установки текста на кнопку используется параметр value. На графическую кнопку пожно установить свое изображение с помощью параметра src, width и height установят размер изображения, alt - указать альтернативный текст.

Остальные элементы ввода

Дата-лист:
code: <input type="text" list="opt"><datalist id="opt"><option value="раз"><option value="два"><option value="три"></datalist>
Филдсет:
Логин:

code: <fieldset><legend>Логин:</legend><input type="text"><input type="password"><input type="submit"></fieldset>
Селект:
code: <select><option value="0" disabled selected>Не выбрано</option><option value="1">Одын</option><option value="2">Дува</option><option value="3">Тры</option></select>
Textarea:
code: <textarea name="nm" id="" cols="70" rows="4" wrap="hard"></textarea>
Дата-лист: применяется к обычному текстовому инпуту, содержит в себе некоторый список значений, которые предлагаются при вводе текста в инпут. Задается при помощи указания значения id атрибута дата-листа в качестве параметра атрибута list у инпута.
Филдсет: делает обрамление вокруг содержимого, можно использовать для группирования инпутов, да и любого другого контента, элемент legend задает заголовок у этого обрамления (см. код примера). Атрибут disabled запрещает использование элементов внутри филдсета.
Селект: комбобокс, значения задаются через option, опции можно указать атрибут disabled, тогда это значение будет недоступным, атрибут selected установит значение по умолчанию. Значения на сервер передаются по name селекта.
Textarea: мультистрочная, атрибут wrap отвечатет за перенос строк, значение soft значит, что переносы, отображаемые в инпуте, не будут отправлены на сервер, данные уйдут одной строкой, hard значит, что переносы сохранятся, при этом обязателен атрибут cols, значение off выключает переносы, текст будет вводиться одной строкой. Атрибут disabled отключает поле.
